##  Feature Voting

TC10 uses a **queue-based voting system**, not a knockout system.

1. New features are proposed via issues
2. Community votes (3 votes per person)
3. Top-voted features get built first
4. Unselected features stay in the queue — **nobody gets eliminated**

> **No winner-takes-all. Just first-come, first-served.**
